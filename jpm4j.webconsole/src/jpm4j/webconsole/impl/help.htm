Help for JPM4J

This JPM Web Console plugin provides a convenient way to find and install bundles. It finds its bundles through jpm4j, a large catalog of Java binaries. Bundles can be found by searching on keywords or (when the search bar is empty) on Web Console plugins.

1 Press this button to activate the JPM Manager.
2 The search bar. Enter a search term. Leaving the field empty will search for plugins that are specifically marked for the Web Console. You can enter modifiers that limit the search. A modifier is a word or letter, then a colon (':') and then a parmater. The following modifiers are available:
  • a: - Artifact id, for example a:org.osgi
  • g: - Group id, for example a:org.osgi
  • p: - Package name, for example p:org.osgi.service.cm
  • bsn: - The bundle symbolic name, for example bsn:org.apache.felix.*
  • cat: - Category, for example cat:test
  • version: - Match the version literally (required for Maven sometimes), for example version:1.0.04-SNAPSHOT
  • baseline: - Match the non-qualifier part of the version in normalized form, for example baseline:1.0.4
  • qualifier: - Match the qualifier part of the version, for example qualifier:2012*
  • o: - Match the organization name, for example o:aQute*
3 The search button. Press to search.
4 The help button, brings you on this page.
5 The place where results are shown. The first symbol signifies the state of the bundle. The following symbols are used:
  • Is installed, can be managed through the bundle tab.
  • Is not installed currently but can be downloaded and installed.
  • Installed but there is a new version available.
6 A button that provides the most logical action for this bundle:
  • Manage - Bundle is installed and correct version, the button will take you to the bundles tab.
  • Install - Bundle is not installed, the button will attempt to install this bundle. It is possible that this will not succeed, see [7] for more the status
  • Update - The bundle is installed but there is a newer version. Press the button to update the bundle.
7 Status of the last command or empty when it was ok. The X button can clear the line if it contains a message.

Configuration

This plugin can be configured with Config Admin, its PID is:

jpm4j.webconsole.impl.WebconsoleRemotePlugins

The following configuration fields can be set:

Field Type Default Description
url URI https://jpm4j.org/rest The URL to the JPM library
debug BOOL false Will print out debugging information to the log

For Webconsole Developers

Web Console Developers that want to show up in the default search list should add a x-jpm-plugin OSGi capability to their manifest with an attribute of x-jpm-plugin=apache.felix.webconsole . For example: 

	Provide-Capability: \
		x-jpm-plugin; \
			x-jpm-plugin=apache.felix.webconsole

All bundles in Maven Central and many other repositories are automatically added. You can also maintain your own repository, potentially in addition to Maven Central, on Github, see how to add a repository to jpm4j.
